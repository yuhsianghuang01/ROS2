🚀 快速開始

1帳號設定與註冊

📝 註冊步驟：
  1. 前往 Google Cloud Console
  2. 使用 Google 帳號登入（沒有的話需要先註冊）
  3. 同意服務條款
  4. 選擇國家/地區（台灣）
  5. 驗證手機號碼
  6. 綁定信用卡（新用戶可獲得 $300 免費額度）
💡 新用戶優惠：
  • $300 美元免費額度：可使用 90 天
  • Always Free 層級：部分服務永久免費
  • Compute Engine：每月 1 個 f1-micro 執行個體免費
  • Cloud Storage：每月 5GB 標準儲存免費

2控制台介面介紹

🎛️ 主要介面組件：
  • 導航選單：左側的漢堡選單，包含所有服務
  • 專案選擇器：頂部的專案下拉選單
  • 搜尋列：快速找到服務和資源
  • 通知中心：右上角的鈴鐺圖示
  • 使用者設定：右上角的個人頭像
  • Cloud Shell：右上角的終端圖示

3建立第一個專案

# 透過 Cloud Shell 建立專案
gcloud projects create my-first-project --name="我的第一個專案"

# 設定為目前專案
gcloud config set project my-first-project

# 查看專案資訊
gcloud projects describe my-first-project
✅ 專案建立成功後，您可以：
  • 啟用所需的 API 服務
  • 建立和管理資源
  • 設定權限和存取控制
  • 監控使用量和費用

4計費設定

💳 計費帳戶設定步驟：
  1. 前往「計費」頁面
  2. 建立計費帳戶
  3. 輸入信用卡資訊
  4. 將專案連結到計費帳戶
  5. 設定預算提醒
⚠️ 重要提醒：
  • 即使有免費額度，仍需綁定信用卡
  • 務必設定預算提醒避免意外費用
  • 定期檢查使用量報告
  • 不用的資源要及時刪除

🔧 核心服務介紹

⚡ 運算服務

服務名稱 用途 適用場景 計費方式
Compute Engine 虛擬機器 傳統應用、完整控制 按分鐘計費
App Engine 無伺服器應用 Web 應用、自動擴展 按請求計費
Cloud Functions 事件驅動函數 微服務、API 按執行次數計費
Cloud Run 容器化應用 Docker 應用 按請求計費
Kubernetes Engine (GKE) 容器編排 大型應用、微服務 按節點計費

💾 儲存服務

服務名稱 類型 適用場景 特色
Cloud Storage 物件儲存 檔案備份、靜態網站 全球分散、高可用
Persistent Disk 區塊儲存 VM 硬碟、資料庫 高效能、可擴展
Filestore 檔案儲存 共享檔案系統 NFS 相容

🌐 網路服務

主要網路服務：
  • VPC (Virtual Private Cloud)：私有虛擬網路
  • Cloud Load Balancing：負載均衡器
  • Cloud CDN：內容分發網路
  • Cloud Interconnect：專線連接
  • Cloud VPN：VPN 連接
  • Cloud DNS：DNS 服務

🗄️ 資料庫服務

服務名稱 資料庫類型 特色 適用場景
Cloud SQL 關聯式資料庫 MySQL, PostgreSQL, SQL Server 傳統應用
Cloud Spanner 全球分散式 SQL 強一致性、水平擴展 大型企業應用
Firestore NoSQL 文件資料庫 即時同步、離線支援 行動應用、Web 應用
Bigtable NoSQL 寬列資料庫 低延遲、高吞吐量 分析、IoT

🛠️ 實際操作教學

1建立虛擬機器 (Compute Engine)

# 使用 gcloud 建立 VM
gcloud compute instances create my-vm \
    --zone=asia-east1-a \
    --machine-type=e2-micro \
    --subnet=default \
    --image-family=ubuntu-2004-lts \
    --image-project=ubuntu-os-cloud \
    --boot-disk-size=10GB \
    --tags=http-server

# 連接到 VM
gcloud compute ssh my-vm --zone=asia-east1-a

# 查看 VM 列表
gcloud compute instances list
💡 VM 操作技巧：
  • 選擇適當區域：asia-east1 (台灣) 延遲最低
  • 機器類型：e2-micro 適合輕量工作負載
  • 預先配置映像：節省設定時間
  • 設定防火牆規則：控制網路存取

2檔案儲存實作 (Cloud Storage)

# 建立儲存貯體
gsutil mb -p my-first-project -c STANDARD -l asia-east1 gs://my-unique-bucket-name

# 上傳檔案
gsutil cp local-file.txt gs://my-unique-bucket-name/

# 下載檔案
gsutil cp gs://my-unique-bucket-name/local-file.txt ./downloaded-file.txt

# 列出貯體內容
gsutil ls gs://my-unique-bucket-name

# 設定公開存取
gsutil acl ch -u AllUsers:R gs://my-unique-bucket-name/local-file.txt

3部署網站應用 (App Engine)

# 建立應用程式
gcloud app create --region=asia-northeast1

# 建立 app.yaml 設定檔
echo 'runtime: python39' > app.yaml

# 建立簡單的 Python 應用
cat > main.py << 'EOF'
from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, GCP World!'

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=8080, debug=True)
EOF

# 建立 requirements.txt
echo 'Flask==2.0.1' > requirements.txt

# 部署應用
gcloud app deploy

# 開啟應用
gcloud app browse
🎉 恭喜！您已成功：
  • 建立了一個虛擬機器
  • 設定了雲端儲存
  • 部署了第一個 Web 應用

📚 進階主題

🔒 安全最佳實務

🛡️ 關鍵安全措施：
  • 身分與存取管理 (IAM)
    • 使用最小權限原則
    • 定期審查權限
    • 使用服務帳戶
  • 網路安全
    • 設定防火牆規則
    • 使用私人 IP
    • 啟用 VPC Flow Logs
  • 資料保護
    • 啟用靜態加密
    • 使用 Cloud KMS
    • 定期備份資料
# IAM 權限管理範例
# 為使用者授予檢視權限
gcloud projects add-iam-policy-binding my-first-project \
    --member="user:user@example.com" \
    --role="roles/viewer"

# 建立服務帳戶
gcloud iam service-accounts create my-service-account \
    --description="My service account" \
    --display-name="My Service Account"

# 產生服務帳戶金鑰
gcloud iam service-accounts keys create ~/key.json \
    --iam-account=my-service-account@my-first-project.iam.gserviceaccount.com

💰 成本優化

💡 節省成本的策略：
  • 選擇適當的機器類型
    • 使用預搶式執行個體 (最多可省 80%)
    • 考慮承諾使用折扣
    • 定期檢視建議
  • 儲存成本優化
    • 選擇適當的儲存類別
    • 設定生命週期政策
    • 刪除不需要的快照
  • 監控與提醒
    • 設定預算提醒
    • 使用計費導出功能
    • 定期審查使用量
# 建立預算提醒
gcloud beta billing budgets create \
    --billing-account=BILLING_ACCOUNT_ID \
    --display-name="My Budget" \
    --budget-amount=100USD \
    --threshold-rule=percent:50 \
    --threshold-rule=percent:90

# 查看計費資訊
gcloud billing accounts list
gcloud billing projects list --billing-account=BILLING_ACCOUNT_ID

📊 監控與日誌

🔍 監控工具：
  • Cloud Monitoring：效能監控和提醒
  • Cloud Logging：集中化日誌管理
  • Cloud Trace：分散式追蹤
  • Cloud Profiler：應用程式效能分析
# 查看日誌
gcloud logging read "resource.type=gce_instance" --limit=10

# 建立提醒政策
gcloud alpha monitoring policies create --policy-from-file=policy.yaml

# 查看監控指標
gcloud monitoring metrics list --filter="metric.type:compute"

🔧 故障排除指南

❌ 常見問題與解決方案

問題 1：無法建立執行個體
可能原因：
  • 配額不足
  • API 未啟用
  • 權限不足
解決方案：
  • 檢查配額限制：gcloud compute project-info describe
  • 啟用 Compute Engine API
  • 確認 IAM 權限
問題 2：無法連接到執行個體
可能原因：
  • 防火牆規則阻擋
  • SSH 金鑰問題
  • 執行個體未啟動
解決方案：
  • 檢查防火牆規則：gcloud compute firewall-rules list
  • 重新產生 SSH 金鑰
  • 確認執行個體狀態
問題 3：帳單費用過高
預防措施：
  • 設定預算提醒
  • 定期檢查計費報告
  • 刪除不使用的資源
  • 使用預搶式執行個體
