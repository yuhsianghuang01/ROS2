iNAD 系統教育訓練規劃表

✅ 一、背景說明

本次委託 A公司 針對 iNAD 系統進行完整教育訓練，主要目的在於：

  • 提升內部人員對 iNAD 系統架構的理解
  • 熟悉各功能模組的操作與維護方式
  • 建立未來系統優化與整合的基礎知識庫

委託單位：台灣大資源服務技術處／工務暨運籌系統部／服務技術課
受訓對象：相關工程師、系統管理員、專案成員等
預估時數：72 小時（共 6 個月，每月 4 週，每週 3 小時）

🧾 二、訓練項目整理（依據來信內容）

項次 教育項目 說明
1 系統基礎介紹 不在 152 個模組內的相關功能
2 功能模組內容說明（程式） 含以下子項目：
1. 系統維護管理
2. 告警查詢管理
3. 一般查詢管理
4. 告警設備關聯資訊查詢
5. 全區輪值排班管理系統
6. 機房日誌程式
7. 網管中心作業管理
8. Paging 設定管理
9. 障礙管理及處理系統
10. IFM Presentation Layer
11. 手動告警簡訊管理
12. Splunk for ISMS 管理
13. 公共工程影響通報機制
3 系統維護與管理說明 系統權限配置、帳號管理、維護流程等
4 資料庫結構說明 DB Table 架構與各功能模組之間的關聯
5 告警收容機制說明 log 傳輸機制、格式、種類、記錄方式、是否涉及機敏資料需隱碼等

🛠️ 三、建議執行方式（快速有效）

🔹 Step 1：建立教育訓練綱要（SOP 文件）

  • 制定一份「iNAD 教學大綱」文件，依序列出每週主題與講師安排。
  • 可搭配 Google Sheet 或 Notion 作為協同平台。

🔹 Step 2：分階段教學（每週聚焦一個主軸）

月份 教學重點 建議方式
第1月 系統基礎 + 功能模組概覽 結合簡報 + 畫面導覽
第2月 功能模組深入說明（1~6項） 工程師現場 coding / Demo
第3月 功能模組深入說明（7~13項） 工程師現場 coding / Demo
第4月 系統維護管理 + 權限架構 系統管理員分享經驗
第5月 資料庫結構說明 DBA 解說 Schema + ER Diagram
第6月 告警收容機制 + 總結 Q&A 結合 Log 分析工具展示 + QA

🔹 Step 3：製作教材與錄影存檔

  • 每場訓練皆須有：
    • 投影片（PPT / PDF）
    • 畫面操作錄製（可使用 OBS、Zoom 錄影）
    • 簡易筆記或操作手冊（供後續查閱）

🔹 Step 4：建立問答與問題追蹤表

  • 使用 Google Sheet 建立：
    • 問題描述
    • 提出人
    • 回覆人
    • 處理狀態（待處理 / 已回覆 / 需進一步討論）

🔹 Step 5：定期彙整與盤點優化建議

  • 每月會議中同步進度，並討論：
    • 哪些功能未來新系統需保留？
    • 哪些功能可以優化整合？
    • 是否有重複/冗餘模組？

📦 四、推薦工具清單（提升效率）

工具名稱 用途
Google Drive 上傳投影片、錄音檔、教材
Zoom / Teams 線上會議 + 錄影
Notion / Trello 協作管理與進度追蹤
OBS Studio 高品質螢幕錄影
Draw.io / Lucidchart 畫 ER Diagram、流程圖
GitLab / GitHub 存放部分程式碼片段供參考（如有需要）

🎯 五、其他建議事項

  1. 建議邀請原開發團隊參與教學
    讓最了解系統的人直接說明，減少誤解。
  2. 鼓勵學員提出優化建議
    在每次課程結束後開放提問與建議，為未來系統升級做準備。
  3. 搭配沙盒環境練習
    若有測試環境，讓學員實際操作，強化學習效果。
  4. 設定學習目標與成果驗收
    每階段可設小考或實作任務，確保吸收成效。
