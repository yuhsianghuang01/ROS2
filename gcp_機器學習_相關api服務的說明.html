如何呼叫 GCP 的 Cloud Speech-to-Text API

以下是一個簡單的範例程式，示範如何使用 Google Cloud Platform (GCP) 的 Cloud Speech-to-Text API 來進行語音轉文字（Speech-to-Text）的請求。這個範例使用 curl 呼叫 REST API，並假設你已經設定好 GCP 專案與服務帳號金鑰。

✅ 前置準備

  • 啟用 Speech-to-Text API：請至 Google Cloud Console 啟用 Speech-to-Text API。
  • 建立服務帳號金鑰：取得 JSON 格式的服務帳號金鑰檔，例如：your-service-account.json。
  • 安裝 gcloud CLI 工具（可選）：可以使用 gcloud auth application-default print-access-token 來取得授權 token。
  • 準備語音檔案：確保有一個 .wav 或 .flac 音訊檔案，例如：audio.raw（建議為 PCM 16kHz 16-bit mono）。

🧪 測試範例：使用 curl 呼叫 Speech-to-Text API

# 1. 取得 access token（需要先登入 gcloud）
ACCESS_TOKEN=$(gcloud auth application-default print-access-token)

# 2. 定義 API endpoint
API_URL="https://speech.googleapis.com/v1/speech:recognize"

# 3. 建立 JSON 請求內容
JSON_REQUEST='{
"config": {
"encoding": "LINEAR16",
"languageCode": "en-US",
"sampleRateHertz": 16000
},
"audio": {
"content": "'$(base64 -w0 audio.raw)'"
}
}'

# 4. 發送 POST 請求
curl -X POST \
-H "Authorization: Bearer $ACCESS_TOKEN" \
-H "Content-Type: application/json; charset=utf-8" \
-d "$JSON_REQUEST" $API_URL

🔍 請求說明：

  • encoding：音訊格式，此處使用 LINEAR16（PCM 16-bit）。
  • languageCode：語言代碼，如 en-US 或 zh-TW。
  • sampleRateHertz：音訊取樣率，需符合音檔規格（常見為 16000 Hz）。

📝 結果範例（回應）：


{
  "results": [
    {
      "alternatives": [
    {
      "transcript": "Hello Google Cloud, this is a test.",
      "confidence": 0.95
    }
      ]
    }
  ]
}

📌 注意事項：

  • 如果你沒有安裝 gcloud CLI，也可以手動產生 OAuth 2.0 Access Token 或使用 Service Account Key 直接簽發 JWT。
  • 若語音檔太大（超過 1MB），建議改用 非同步請求（speech:longrunningrecognize）。
  • 支援語言請參閱官方文件：Cloud Speech-to-Text 支援語言列表。
