☁️ Cloud Shell vs Cloud Shell CLI 差別說明

🌐 Cloud Shell（雲端殼層）

📝 定義：

Cloud Shell 是一個基於瀏覽器的完整 Linux 開發環境，由 Google 免費提供。

🎯 主要特色：
  • 完整的虛擬機器：Debian Linux 環境
  • 5GB 永久儲存空間：$HOME 目錄會保存
  • 預安裝工具：gcloud、kubectl、docker、git 等
  • 內建程式碼編輯器：類似 VS Code 的網頁編輯器
  • 網頁預覽功能：可預覽 Web 應用
  • 免費使用：每週 50 小時免費額度
🔧 如何啟動：
# 在 GCP Console 右上角點擊終端圖示
# 或前往：https://shell.cloud.google.com/

⚡ Cloud Shell CLI（命令列介面）

📝 定義：

Cloud Shell CLI 是指在 Cloud Shell 環境中使用的命令列工具集合。

🛠️ 包含的工具：
  • gcloud CLI：Google Cloud 主要命令工具
  • gsutil：Cloud Storage 操作工具
  • kubectl：Kubernetes 管理工具
  • bq：BigQuery 命令列工具
  • terraform：基礎設施即程式碼工具
  • docker：容器管理工具
💡 常用命令範例：
# gcloud CLI 範例
gcloud projects list
gcloud compute instances list

# gsutil 範例
gsutil ls
gsutil cp file.txt gs://bucket-name/

# kubectl 範例
kubectl get pods
kubectl apply -f deployment.yaml

🎯 實際使用場景

🌐 使用 Cloud Shell 的情況：

  • 需要完整的開發環境
  • 要編輯較大的程式碼專案
  • 需要圖形化介面操作
  • 要預覽 Web 應用程式
  • 進行多工作並行

⚡ 使用 Cloud Shell CLI 的情況：

  • 快速執行 Google Cloud 命令
  • 自動化腳本執行
  • 簡單的資源管理操作
  • CI/CD 管道中的命令
  • 一次性的管理任務

🔗 兩者的關係

Cloud Shell CLI 是 Cloud Shell 環境的一部分：

  • 當您啟動 Cloud Shell 時，所有 CLI 工具都已經預安裝好了
  • 您可以在 Cloud Shell 的終端中使用所有的 CLI 命令
  • Cloud Shell 提供了使用 CLI 工具的最佳環境
  • 不需要額外安裝或設定，開箱即用

📚 學到的知識點

  • Cloud Shell 是一個基於瀏覽器的完整 Linux 開發環境。
  • Cloud Shell CLI 是在 Cloud Shell 環境中使用的命令列工具集合。
  • gcloud CLI 是 Google Cloud 的主要命令工具。
  • SKU (Stock Keeping Unit) 是 GCP 中的計費單位，每個可計費項目都有唯一的 SKU。
  • SKU 分組是將相關 SKU 組織在一起的邏輯分類，用於簡化計費管理。
  • 了解 SKU 有助於成本優化和精確的預算規劃。
  • 可以使用 gcloud billing skus list 命令查看所有可用的 SKU。
  • 不同區域的相同服務可能有不同的 SKU 定價。

💡 常用命令範例

# 列出所有專案
gcloud projects list

# 列出所有 VM 實例
gcloud compute instances list

# 查看所有可用的 SKU
gcloud billing skus list

# 查看特定服務的 SKU
gcloud billing skus list --filter="serviceDisplayName:Compute Engine"

# 查看計費帳戶
gcloud billing accounts list

# 查看專案的計費資訊
gcloud billing projects describe PROJECT_ID

⚠️ 重要注意事項

  • 會話超時：Cloud Shell 會話在 20 分鐘不活動後會終止
  • 使用限制：每週 50 小時的免費使用時間
  • 網路存取：某些網路環境可能限制存取
  • 效能限制：適合輕到中等的工作負載
  • 資料保存：只有 $HOME 目錄的資料會永久保存

🎯 總結

  • Cloud Shell = 完整的雲端開發環境（包含終端、編輯器、預覽等）
  • Cloud Shell CLI = 在該環境中使用的命令列工具集合
  • 簡單來說，Cloud Shell 是「房子」，而 Cloud Shell CLI 是「房子裡的工具」
  • 當您使用 Cloud Shell 時，所有的 CLI 工具都已經準備好讓您使用了！

💰 SKU 和 SKU 分組詳細說明

📝 什麼是 SKU？

📖 SKU 定義：

SKU (Stock Keeping Unit) 在 GCP 中是指計費單位，代表特定服務的可計費項目。每個 SKU 都有唯一的識別碼和計費規則。

🎯 SKU 的主要特徵：
  • 唯一識別：每個 SKU 都有唯一的 ID
  • 服務特定：綁定到特定的 GCP 服務
  • 計費基礎：定義如何計算使用量和費用
  • 區域相關：可能因地理區域而有不同定價
  • 時間敏感：計費方式可能隨時間變化
💡 SKU 範例：
# Compute Engine VM 的 SKU 範例
# n1-standard-1 機器在 us-central1 區域的 CPU 使用量
SKU ID: 6F81-5844-456A
Description: "N1 Predefined Instance Core running in Americas"

# Cloud Storage 的 SKU 範例
# Standard 儲存在 us-central1 的每月儲存費用
SKU ID: 95FF-2EF5-5EA1
Description: "Standard Storage US Regional"

📊 什麼是 SKU 分組？

📖 SKU 分組定義：

SKU 分組 (SKU Groups) 是將相關的 SKU 組織在一起的邏輯分類，用於簡化計費管理和成本分析。

🗂️ 常見的 SKU 分組：
  • 按服務分組：Compute Engine、Cloud Storage、BigQuery
  • 按資源類型分組：CPU、記憶體、儲存、網路
  • 按地理區域分組：美洲、歐洲、亞太地區
  • 按使用模式分組：按需計費、承諾使用折扣、預搶式
📋 SKU 分組範例：
# 查看特定服務的 SKU
gcloud billing skus list --service=services/6F81-5844-456A

# 按服務篩選 SKU
gcloud billing skus list --filter="serviceDisplayName:Compute Engine"

# 查看所有可用服務
gcloud billing services list

🔍 SKU 的詳細結構

SKU 屬性 說明 範例
SKU ID 唯一識別碼 6F81-5844-456A
Name SKU 名稱 N1 Predefined Instance Core
Description 詳細描述 N1 Predefined Instance Core running in Americas
Category 資源類別 Compute、Storage、Network
Service 所屬服務 Compute Engine、Cloud Storage
Pricing Info 計費資訊 每小時 $0.0475 USD

🛠️ 如何查看和管理 SKU

📋 常用 SKU 管理命令：

# 列出所有可用的 SKU
gcloud billing skus list

# 搜尋特定服務的 SKU
gcloud billing skus list --filter="serviceDisplayName:Compute Engine"

# 查看特定 SKU 的詳細資訊
gcloud billing skus describe SKU_ID

# 按區域篩選 SKU
gcloud billing skus list --filter="serviceRegions:us-central1"

# 查看 SKU 的定價資訊
gcloud billing skus list --filter="name:*standard*" --format="table(name,description,category,serviceRegions[0]:label=REGION)"

💰 實用的 SKU 查詢範例：

# 查看 Compute Engine 所有機器類型的 SKU
gcloud billing skus list \
    --filter="serviceDisplayName:'Compute Engine' AND category:'Compute'" \
    --format="table(name,description,category,serviceRegions[0])"

# 查看 Cloud Storage 的所有 SKU
gcloud billing skus list \
    --filter="serviceDisplayName:'Cloud Storage'" \
    --format="table(name,description,category)"

# 查看特定區域的所有 SKU
gcloud billing skus list \
    --filter="serviceRegions:asia-east1" \
    --format="table(name,description,serviceDisplayName)"

💡 SKU 使用的最佳實務

  • 定期檢查 SKU 變更：Google 可能會調整定價或新增 SKU
  • 了解計費模式：不同 SKU 有不同的計費週期（秒、分鐘、小時）
  • 區域選擇影響：同樣的服務在不同區域可能有不同的 SKU 定價
  • 承諾使用折扣：長期使用可透過 CUD (Committed Use Discounts) 節省成本
  • 監控使用量：設定預算提醒避免意外費用

🎯 為什麼要了解 SKU？

  • 成本優化：了解每個資源的實際計費方式
  • 預算規劃：精確預估專案所需費用
  • 資源選擇：選擇最符合需求且成本效益高的資源
  • 費用分析：深入分析哪些服務產生最多費用
  • 自動化管理：透過 API 自動監控和管理計費
